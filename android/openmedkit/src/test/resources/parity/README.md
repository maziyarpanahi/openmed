# Android Span Parity Fixtures

`android_span_parity.json` is the golden resource for Android-side
token-classification parity tests. It is generated by
`scripts/android/generate_parity_fixtures.py` from an Android-profile ONNX
export and contains only synthetic, PHI-free note text.

Android tests should load:

- `cases[].text` as the exact model input.
- `cases[].tokens[].id` as the expected tokenizer IDs.
- `cases[].tokens[].offset` as exact `[start, end)` UTF-16/Unicode scalar
  character offsets into `cases[].text`.
- `cases[].spans[]` as the Python reference spans. Span boundaries are exact:
  `contract.span_boundaries.tolerance_chars` is `0`.

The tolerance contract is intentionally strict:

- token IDs must match exactly;
- character offsets must match exactly;
- span canonical labels must match exactly;
- span `start` and `end` boundaries must match exactly;
- logit ties use the lowest label ID so Python and Android stay deterministic.

Span records do not store surface text. Compare the Android span offsets against
`cases[].text` and use `text_hash` only as a privacy-preserving integrity check.
The full protocol is documented in `docs/android-parity.md`.
